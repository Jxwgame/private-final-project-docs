# Monitoring & Audit Notes — แนวคิดการติดตามและตรวจสอบระบบ

เอกสารนี้สรุปแนวคิดด้านการบันทึกเหตุการณ์ (Logging),
การตรวจสอบ (Audit) และการติดตามสถานะระบบ (Monitoring)
ในระดับการออกแบบเชิงระบบ โดยมุ่งเน้นความพร้อมในการปฏิบัติการ
และการตอบสนองต่อเหตุการณ์ด้านความมั่นคงปลอดภัย

---

## เป้าหมายของการ Monitoring

- เพิ่มความสามารถในการมองเห็นพฤติกรรมของระบบ
- รองรับการตรวจสอบย้อนหลังเมื่อเกิดเหตุการณ์ผิดปกติ
- ลดระยะเวลาในการตรวจจับและตอบสนองต่อเหตุการณ์ด้านความปลอดภัย
- สนับสนุนการปฏิบัติงานของทีมระบบและทีมความปลอดภัย

---

## แยกประเภทเหตุการณ์ที่ควรบันทึก (เชิงแนวคิด)

- การยืนยันตัวตนและการเข้าใช้งานระบบ (สำเร็จ / ล้มเหลว)
- การเปลี่ยนแปลงสิทธิ์หรือบทบาทของผู้ใช้
- การเข้าถึงทรัพยากรหรือข้อมูลสำคัญ
- การเรียกใช้งาน API ที่ผิดปกติหรือเกินอัตราที่กำหนด
- การทำงานผิดพลาดของระบบหรือ service หลัก

> หมายเหตุ: หลีกเลี่ยงการบันทึกข้อมูลอ่อนไหวโดยตรงใน log

---

## แนวคิดการออกแบบ Logging

- ใช้ **Structured Logging** เพื่อให้สามารถค้นหาและวิเคราะห์ได้ง่าย
- กำหนดรูปแบบข้อมูล log ให้สอดคล้องกันทุก service
- รองรับการเชื่อมโยงเหตุก
